# 🎮 Бот поиска тиммейтов — CS2 / Dota 2

Telegram-бот для поиска напарников по играм (аналог «Леонардо Дайвинчик», но
заточен под две игры). У каждой игры — **своя отдельная анкета**: данные CS2 и
Dota 2 не смешиваются. Лайки, лайк с сообщением, пропуск, жалобы и взаимная
симпатия с обменом контактами работают «из коробки».

---

## ✨ Возможности

- Выбор игры при входе: **Counter-Strike 2** или **Dota 2**.
- Создание анкеты: никнейм, пол, возраст, ранг (FACEIT Elo / MMR), описание, фото.
- Одна анкета на игру на пользователя — данные не двоятся.
- Лента анкет с оценкой совпадения по рангу («🎯 Идеальное совпадение» … «📉 Значительная разница»).
- ❤️ Лайк, 💌 лайк + сообщение, ➡️ пропуск, 🚫 жалоба.
- Уведомление «твоя анкета понравилась» и **взаимная симпатия** с контактами.
- Редактирование анкеты: имя, возраст, описание, ранг, пол, фото; пересоздание и удаление.
- Автоскрытие анкеты после N жалоб (настраивается).
- Чистая навигация без конфликтов кнопок (см. ниже).

## 🚀 Быстрый старт

Нужен **Python 3.11+**.

```bash
# 1. Перейти в папку проекта
cd cs2_dota_teammates_bot

# 2. Создать виртуальное окружение
python -m venv .venv
# Windows:
.venv\Scripts\activate
# macOS / Linux:
source .venv/bin/activate

# 3. Установить зависимости
pip install -r requirements.txt

# 4. Настроить переменные окружения
cp .env.example .env        # Windows: copy .env.example .env
#  → открой .env и впиши BOT_TOKEN от @BotFather

# 5. Запустить бота
python bot.py
```

Готово — напиши боту `/start`.

> **⚠️ Нужен Python 3.10+** (в коде используется синтаксис `str | None`, который
> на 3.9 и ниже не работает). Если системный `python` старый — самый простой путь
> поднять нужную версию без прав администратора:
> ```bash
> # установит uv (если его нет) и через него Python + окружение
> curl -LsSf https://astral.sh/uv/install.sh | sh
> uv venv --python 3.12 .venv
> uv pip install --python .venv/bin/python -r requirements.txt
> .venv/bin/python bot.py
> ```

## 🗄️ База данных

По умолчанию используется **SQLite** (файл `bot.db` создаётся сам, настройка не
нужна) с включённым режимом WAL — этого достаточно для тысяч анкет.

Для высокой нагрузки переключись на **PostgreSQL** без изменения кода:

1. Раскомментируй `asyncpg` в `requirements.txt` и установи: `pip install asyncpg`.
2. В `.env` укажи строку подключения:
   ```
   DATABASE_URL=postgresql+asyncpg://user:password@localhost:5432/teammates_bot
   ```
3. Запусти бота — таблицы создадутся автоматически.

Схема (3 таблицы) с нужными индексами под «горячие» запросы:

- `users` — пользователи Telegram (для обмена контактами).
- `profiles` — анкеты, `UNIQUE(user_id, game)` → одна анкета на игру.
- `interactions` — лайки/пропуски/жалобы, `UNIQUE(actor_id, target_id, game)` →
  анкета не показывается повторно; индексы ускоряют ленту и проверку взаимности.

## 🛡️ Почему не конфликтуют кнопки

- **Навигация — reply-клавиатура**, её роутер (`start.py`) подключён первым,
  поэтому кнопки «Главное меню» / «Отмена» / выбор игры срабатывают раньше любого
  ввода в анкете — из любого состояния можно выйти.
- **Действия в ленте — inline-кнопки**, и в `callback_data` зашиты игра и id цели,
  так что каждое нажатие однозначно.
- Лента держит **одну активную карточку**: перед показом следующей анкеты
  предыдущая удаляется — «устаревших» кнопок не остаётся.

## ⚙️ Настройки (`.env`)

| Переменная        | Описание                                            | По умолчанию              |
|-------------------|-----------------------------------------------------|---------------------------|
| `BOT_TOKEN`       | Токен от @BotFather (обязательно)                   | —                         |
| `DATABASE_URL`    | Строка подключения к БД                             | `sqlite+aiosqlite:///bot.db` |
| `ADMIN_IDS`       | ID админов через запятую                            | пусто                     |
| `REPORTS_TO_HIDE` | Сколько жалоб скрывают анкету                        | `5`                       |
| `REBROWSE_COOLDOWN_MINUTES` | Через сколько минут лайкнутая/пропущенная анкета снова попадётся в ленте | `60` |

## 📈 Заметки по масштабированию

- Лента сейчас использует `ORDER BY random()` — отлично для тысяч анкет.
  Для десятков тысяч+ замени на курсорную пагинацию по `id`.
- FSM-состояния хранятся в памяти. Для нескольких процессов/перезапусков без потери
  незавершённых анкет подключи Redis-хранилище (`aiogram` его поддерживает).
